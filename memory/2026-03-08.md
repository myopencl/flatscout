### Issue with `flatscout-scraper-api` and `customUrl`
- **Problem:** The `--customUrl` parameter in `scripts/create-search.js` does not function as described in `SKILL.md`. When a custom URL is provided, the API ignores the full URL and generates a simplified `searchUrl` based on basic filters (city, operation, property type).
- **Impact:** Advanced geographical filters (e.g., `distanceRadius`, `placeId`, `mapBounds`, `district_id`) from the original custom URLs are not applied to the saved searches.
- **Current Status:** Verified by creating two searches (Otodom and OLX) with `customUrl`s, both resulted in simplified `searchUrl`s.
- **Resolution Plan:**
    1. Delete current incorrect searches.
    2. Recreate searches by explicitly passing all filter parameters (`city`, `rooms`, `minPrice`, `maxPrice`, `minArea`, `maxArea`, `radiusKm`, `placeId`, `districtId`, etc.) instead of relying on `--customUrl`.
